# My-Speak 产品需求文档 (PRD)

## 1. 项目概述

### 1.1 项目背景
My-Speak 是一个类似 Discord/TeamSpeak 的语音和文字交流软件，专注于为小型社区（20人以内）提供高质量的实时语音通讯体验。

### 1.2 目标用户
- 游戏小队/公会
- 小型学习小组
- 远程工作团队
- 朋友/家庭群组

### 1.3 核心价值
- **低延迟语音**: WebRTC技术实现 <300ms 端到端延迟
- **简单易用**: 无需复杂配置，开箱即用
- **频道分离**: 文字和语音频道独立管理
- **隐私安全**: 端到端加密，一次性邀请码机制

## 2. 功能需求

### 2.1 用户系统

#### 2.1.1 用户注册与登录
- **注册**:
  - 用户名（唯一，3-32字符，仅允许字母、数字、下划线）
  - 邮箱（唯一，需要验证）
  - 密码（8-128字符，必须包含大小写字母和数字）
  - 可选：头像上传（支持JPG/PNG，最大5MB）
  
- **登录**:
  - 邮箱+密码登录
  - JWT Token认证（Access Token + Refresh Token）
  - Token有效期：Access Token 15分钟，Refresh Token 7天
  - 支持"记住我"功能（延长Refresh Token至30天）

- **登出**:
  - 清除本地Token
  - 服务器端标记Token失效（加入黑名单）

#### 2.1.2 用户资料管理
- 修改显示名称
- 修改头像
- 修改密码（需要验证旧密码）
- 查看加入的服务器列表

### 2.2 服务器系统

#### 2.2.1 服务器创建
- 服务器名称（1-100字符）
- 可选描述（最多500字符）
- 可选图标（JPG/PNG，最大5MB）
- 创建者自动成为Owner
- 默认创建"general"文字频道和"General"语音频道

#### 2.2.2 服务器管理
- Owner权限：
  - 修改服务器信息（名称、描述、图标）
  - 删除服务器（级联删除所有频道和消息）
  - 转让所有权
  - 管理成员（提升为Admin、移除成员）
  - 创建邀请码
  - 创建/编辑/删除频道

- Admin权限：
  - 管理成员（仅移除普通成员）
  - 创建/编辑/删除频道
  - 创建邀请码

- Member权限：
  - 加入文字频道查看消息
  - 加入语音频道通话
  - 发送文字消息（文字频道）

#### 2.2.3 加入服务器
- 通过邀请码加入（一次性使用）
- 邀请码格式：8位字母数字混合（如：A1B2C3D4）
- 加入后成为Member角色

#### 2.2.4 离开服务器
- Member可自行离开
- 离开后立即断开所有频道连接
- Owner不能离开，必须先转让所有权或删除服务器

### 2.3 频道系统

#### 2.3.1 频道类型
1. **文字频道 (Text Channel)**
   - 实时消息收发
   - 消息历史记录（加载最近100条，支持滚动加载更多）
   - 支持文本、表情符号
   - 消息时间戳
   - 显示发送者头像和名称

2. **语音频道 (Voice Channel)**
   - 多人实时语音通话
   - WebRTC传输（Opus编码）
   - 可调节比特率（默认64kbps，范围16-128kbps）
   - 用户限制（默认20人）
   - 静音/取消静音
   - 语音活动检测（说话指示器）
   - 音量指示条

#### 2.3.2 频道管理
- 创建频道：
  - 选择类型（文字/语音）
  - 设置名称
  - 语音频道可设置用户限制和比特率
  - 设置位置（排序）

- 编辑频道：
  - 修改名称
  - 修改位置
  - 语音频道可修改用户限制和比特率

- 删除频道：
  - 仅Owner/Admin可删除
  - 删除语音频道时，所有在线用户自动断开
  - 删除文字频道时，所有消息被删除（级联）

### 2.4 邀请码系统

#### 2.4.1 生成邀请码
- 权限：Owner/Admin
- 邀请码属性：
  - 8位随机字符串（字母数字混合，排除易混淆字符0/O/I/l）
  - 最大使用次数（默认1次）
  - 可选过期时间（默认24小时）
  - 创建者记录

#### 2.4.2 使用邀请码
- 用户在界面输入邀请码
- 验证邀请码有效性：
  - 存在且未过期
  - 使用次数未达上限
  - 用户尚未加入该服务器
- 成功后自动加入服务器，邀请码使用次数+1

#### 2.4.3 管理邀请码
- Owner/Admin可查看活跃邀请码列表
- 可手动撤销邀请码（提前失效）
- 查看邀请码使用情况（已用次数/剩余次数）

## 3. 非功能性需求

### 3.1 性能要求

| 指标 | 目标值 | 说明 |
|------|--------|------|
| 语音延迟 | < 300ms | 端到端延迟，包括编码/传输/解码 |
| 页面加载 | < 3s | 首屏加载时间（3G网络） |
| 并发用户 | 20人 | 单频道语音最大并发 |
| 消息实时性 | < 100ms | 消息从发送到显示的时间 |
| 音频质量 | 48kHz | Opus编码采样率 |

### 3.2 安全要求

- **传输加密**: 所有通信使用HTTPS/WSS，WebRTC使用DTLS-SRTP
- **数据加密**: 
  - 密码使用bcrypt加密（成本因子12）
  - JWT Token使用HS256签名
- **输入验证**: 所有用户输入进行长度和格式验证
- **防攻击**:
  - 登录接口：5分钟内最多5次失败尝试，失败后锁定15分钟
  - API限流：每IP每分钟最多100次请求
  - WebSocket连接限流：每IP最多5个并发连接

### 3.3 可用性要求

- **浏览器支持**:
  - Chrome 90+
  - Firefox 90+
  - Safari 14+
  - Edge 90+
- **移动端**: 响应式设计，支持移动浏览器
- **离线提示**: 网络断开时显示重连状态和倒计时
- **错误处理**: 用户友好的错误提示，不暴露技术细节

### 3.4 可扩展性

- 当前设计支持单服务器20人并发
- 架构预留水平扩展能力（通过Redis共享状态）
- 数据库设计支持分片（按server_id分片）

## 4. 用户界面设计

### 4.1 页面结构

```
登录/注册页
    ↓
主应用页
├── 左侧栏：服务器列表
│   ├── 服务器图标（圆形）
│   ├── 当前服务器高亮
│   └── 添加服务器按钮
│
├── 中间栏：频道列表 + 用户信息
│   ├── 服务器名称（顶部）
│   ├── 文字频道区域
│   │   └── 频道项（#图标 + 名称）
│   ├── 语音频道区域
│   │   └── 频道项（🔊图标 + 名称 + 在线人数）
│   └── 用户信息（底部）：头像、名称、静音/设置按钮
│
└── 右侧主区域
    ├── 文字频道视图
    │   ├── 频道名称（顶部）
    │   ├── 消息列表（中间，可滚动）
    │   └── 消息输入框（底部）
    │
    └── 语音频道视图
        ├── 频道名称（顶部）
        ├── 用户网格/列表（中间）
        │   └── 每个用户：头像、名称、静音状态、音量条
        └── 控制栏（底部）：静音按钮、断开按钮、设置
```

### 4.2 关键交互

**加入语音频道流程:**
1. 用户点击语音频道
2. 请求麦克风权限（首次）
3. 初始化WebRTC连接
4. 连接到SFU服务器
5. 开始发送音频流
6. 接收其他用户音频流
7. 显示用户列表和说话状态

**发送文字消息流程:**
1. 用户在输入框输入内容
2. 按Enter或点击发送按钮
3. 消息通过Socket.io实时发送
4. 本地立即显示（乐观更新）
5. 服务器确认后保持显示
6. 其他在线用户实时接收

## 6. 术语表

| 术语 | 说明 |
|------|------|
| Server | 服务器/社区，用户的集合，包含多个频道 |
| Channel | 频道，分为文字频道和语音频道 |
| SFU | Selective Forwarding Unit，选择性转发单元，WebRTC服务器架构 |
| WebRTC | Web实时通信技术，用于浏览器间音视频传输 |
| Socket.io | 实时双向事件通信库 |
| mediasoup | 现代化的WebRTC SFU实现 |
| Opus | 高效的音频编解码器 |
| VAD | Voice Activity Detection，语音活动检测 |
| DTLS | Datagram Transport Layer Security，数据报传输层安全 |
| SRTP | Secure Real-time Transport Protocol，安全实时传输协议 |
| ICE | Interactive Connectivity Establishment，交互式连接建立 |
| SDP | Session Description Protocol，会话描述协议 |
